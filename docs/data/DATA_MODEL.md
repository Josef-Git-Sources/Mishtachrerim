\# DATA\_MODEL.md



Data Model

Project: משתחררים (Mishtachrerim)



This document defines the database structure for the product.



The data model supports:



\- Career paths

\- Courses

\- Quiz questions

\- Quiz answers

\- Users

\- Saved items

\- Checklist items

\- SEO guides

\- Leads



The system should use a relational database.



Recommended DB:

PostgreSQL



---



\# 1. Core Entities



Main entities in the system:



Users  

CareerPaths  

Courses  

QuizQuestions  

QuizAnswers  

QuizResults  

SavedCareers  

SavedCourses  

ChecklistItems  

Guides  

Leads



---



\# 2. Users



Stores registered users.



Users are optional for browsing but required for saving data.



Table: users



Fields:



id (uuid, primary key)



email (string, unique)



name (string)



created\_at (timestamp)



auth\_provider (string)



Example providers:



google  

email



---



\# 3. Career Paths



Stores all career paths used in the system.



Table: career\_paths



Fields:



id (uuid, primary key)



slug (string, unique)



name (string)



hebrew\_name (string)



description (text)



salary\_min (integer)



salary\_max (integer)



training\_time\_months (integer)



created\_at (timestamp)



Example slug values:



qa-tester  

data-analyst  

digital-marketing  

it-support  

project-management  

graphic-design  

ecommerce-manager  

network-technician  

full-stack-developer  

ux-ui-designer



---



\# 4. Courses



Stores curated courses.



Table: courses



Fields:



id (uuid, primary key)



slug (string, unique)



name (string)



provider\_name (string)



duration\_weeks (integer)



learning\_mode (string)



price\_range (string)



editorial\_rating (decimal)



career\_path\_id (uuid, foreign key)



created\_at (timestamp)



learning\_mode examples:



online  

onsite  

hybrid



---



\# 5. Quiz Questions



Stores quiz questions.



Table: quiz\_questions



Fields:



id (uuid, primary key)



question\_text\_he (text)



question\_order (integer)



is\_deep\_quiz\_only (boolean)



created\_at (timestamp)



---



\# 6. Quiz Answers



Stores answer options for each question.



Table: quiz\_answers



Fields:



id (uuid, primary key)



question\_id (uuid, foreign key)



answer\_text\_he (text)



answer\_key (string)



Example answer\_key:



A  

B  

C  

D



---



\# 7. Quiz Scoring



Stores score mapping for each answer.



Table: quiz\_answer\_scores



Fields:



id (uuid, primary key)



answer\_id (uuid, foreign key)



career\_path\_id (uuid, foreign key)



score (integer)



Example:



Answer A →



QA Tester +3  

Data Analyst +2



Each mapping becomes one row.



---



# 8. Quiz Results

Stores quiz attempts by users.

Table: quiz_results

Fields:

id (uuid, primary key)

user_id (uuid, nullable)

created_at (timestamp)

Users can take quiz without login.

If logged out, user_id remains null.

---

# 8.1 Quiz Result Careers

Stores the career matches generated by a quiz result.

Table: quiz_result_careers

Fields:

id (uuid, primary key)

quiz_result_id (uuid, foreign key)

career_path_id (uuid, foreign key)

score (integer)

rank (integer)

Example:

rank 1 → QA Tester
rank 2 → Data Analyst
rank 3 → Digital Marketing

Relationships:

quiz_result_careers.quiz_result_id → quiz_results.id

quiz_result_careers.career_path_id → career_paths.id

# 13.1 Intent Pages

Stores SEO intent pages that target specific search queries.

These pages are designed to capture long-tail search traffic
and connect users directly to relevant career paths.

Table: intent_pages

Fields:

id (uuid, primary key)

slug (string, unique)

title_he (string)

content_markdown (text)

career_path_id (uuid, nullable)

intent_type (string)

created_at (timestamp)

updated_at (timestamp)

intent_type examples:

career-choice
career-comparison
learning-path
salary-question

Example slugs:

should-i-learn-qa
is-data-analyst-good-career
qa-vs-data-analyst
is-digital-marketing-hard

Relationship:

intent_pages.career_path_id → career_paths.id

Purpose:

Allow SEO pages that answer specific user questions
and route users toward career recommendations.


\# 14. Leads



Stores lead submissions.



Table: leads



Fields:



id (uuid, primary key)



name (string)



email (string)



phone (string)



course\_id (uuid)



created\_at (timestamp)



These leads are sent to course providers.



---

---

# 14.1 Analytics Events

Stores product analytics events for tracking user behavior.

Table: analytics_events

Fields:

id (uuid, primary key)

event_name (string)

user_id (uuid, nullable)

session_id (string)

event_metadata (jsonb)

created_at (timestamp)

event_name examples:

quiz_started
quiz_completed
results_viewed
career_page_viewed
course_clicked
lead_submitted

Purpose:

Track key user actions in the decision engine flow.

Example event metadata:

{
  "career_slug": "qa-tester"
}

or

{
  "course_slug": "qa-bootcamp"
}


\# 15. Relationships



Important relations:



courses.career\_path\_id → career\_paths.id



quiz\_answers.question\_id → quiz\_questions.id



quiz\_answer\_scores.answer\_id → quiz\_answers.id



quiz\_answer\_scores.career\_path\_id → career\_paths.id



saved\_careers.user\_id → users.id



saved\_courses.user\_id → users.id



saved\_courses.course\_id → courses.id
